# ai-lint 研发计划

> **版本**: v1.0.0 | **日期**: 2026-07-20 | **总工期**: 2 周

---

## 一、里程碑

| 里程碑 | 时间 | 交付物 | 验收标准 |
|--------|------|--------|---------|
| **M1** | Week 1 末 | `ai-lint` 单文件扫描 | 6 条基础规则可运行 |
| **M2** | Week 2 末 | `ai-lint fix` + 跨文件 + 发布 | npm 发布 + README + GIF |

---

## 二、Week 1：核心扫描引擎

### Day 1：项目骨架 + Parser

```
□ 初始化 TypeScript 项目
□ 实现 Markdown 解析器（parseRules）
□ 实现 YAML frontmatter 解析器（parseFrontmatter）
□ 单元测试（使用真实 fixture）
```

### Day 2：3 条核心规则

```
□ no-duplicate — 字面重复检测 + fix
□ no-verbose — 冗余表述检测 + fix（基于模式字典）
□ max-length — 规则总长度检测
□ 单元测试
```

### Day 3：3 条进阶规则

```
□ no-stale-reference — 死引用检测
□ no-global-path-rule — 误用全局规则检测
□ no-missing-frontmatter — YAML 头缺失检测 + fix
□ 单元测试
```

### Day 4：CLI 骨架 + 渲染

```
□ commander 命令定义
□ 终端输出渲染（chalk，接近 ESLint 风格）
□ 文件发现（自动检测目录中的 AI 配置文件）
□ ai-lint 命令完整可用
```

### Day 5：Week 1 收尾

```
□ 集成测试（真实 CLAUDE.md + 15 个 Skill）
□ 性能测试（扫描 16 个文件 ≤ 1s）
□ 修复 bug
```

**里程碑 M1**: `ai-lint` 可运行

---

## 三、Week 2：修复 + 跨文件 + 发布

### Day 6：fix 命令

```
□ 实现 fixer/deduplicate.ts
□ 实现 fixer/simplify.ts
□ ai-lint fix 命令
□ ai-lint fix --dry-run
```

### Day 7：语义规则 + 跨文件

```
□ no-semantic-duplicate — 基于关键字哈希的语义相似度
□ no-conflict — 跨文件冲突检测
□ no-overlap-skills — 跨 Skill 重叠检测
□ ai-lint --cross-files 命令
```

### Day 8：剩余规则

```
□ no-overconstrain — 过度约束检测
□ no-null-effect — 无效约束检测
□ no-skill-bloat — Skill 体积膨胀检测
```

### Day 9：CI 模式 + 文档

```
□ ai-lint --ci（有问题 exit 1）
□ ai-lint --json 输出
□ README.md（含共生声明 + 示例 GIF）
□ 录制 15 秒 GIF
```

### Day 10：发布

```
□ npm publish
□ V2EX / LINUX DO 发帖
□ 验证 npx ai-lint@1.0.0
```

**里程碑 M2**: v1.0.0 发布
