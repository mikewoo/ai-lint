# ai-lint 项目立项书

> **版本**: v1.0.0 | **日期**: 2026-07-20 | **性质**: 纯利他开源

## 一、项目背景

### 1.1 真实痛点

基于 6 条并行调研流（2026-07-19 ~ 20），从 GitHub Issues、Reddit、社区中挖掘了 22 条真实需求信号：

- **痛点 #1**：$200 plan 70 分钟耗尽，用户不知道 Token 花在哪
- **痛点 #2**：CLAUDE.md 50→400 行后 AI 遵守率从 94% 跌到 71%
- **痛点 #7**：91% 热门 AGENTS.md/CLAUDE.md 含"配置气味"（lint-leakage 62%、context-bloat 42%）

**用户关心的不是配置格式（JSON vs TOML），而是配置的语义健康度** — 有没有冗余、有没有冲突、浪费了多少 Token。

### 1.2 竞品空白

对 57 个竞品进行全面普查，发现：

| 赛道 | 竞品数 | 有无"配置健康检测" |
|------|--------|:---:|
| Token 分析 | 13+ | ❌ |
| Dashboard | 10+ | ❌ |
| 路由/代理 | 10+ | ❌ |
| 压缩/优化 | 9 | ❌ |
| 配置管理 | 7+ | ❌ |
| IDE 内置 | 7+ | ❌ |

**57 个竞品中，0 个在做配置质量检测。** 友商都在做"管理"，没有人做"诊断"。

## 二、项目定位

### 2.1 一句话

> **`npx ai-lint` — 像 ESLint 检查代码一样，检查你的 AI 编程配置是否健康。**

### 2.2 对标模型

```
ESLint → JavaScript 代码质量检测
ai-lint → AI 编程配置健康检测

ESLint                          ai-lint
──────                          ──────
输入  .js / .ts                   CLAUDE.md / AGENTS.md / SKILL.md / .cursorrules
检测  no-unused-vars              no-duplicate-rules / no-conflict / no-skill-bloat
输出  error / warning             问题行号 + Token 浪费估算 + 修复建议
修复  eslint --fix                ai-lint fix
生态  eslint-plugin-xxx           规则市场（未来）
```

### 2.3 不做什么

| 不做 | 原因 |
|------|------|
| ❌ 配置读写管理 | 57 个竞品已在做 |
| ❌ Token 统计 | ccusage 16K Stars 碾压 |
| ❌ Dashboard | 10+ 竞品 |
| ❌ 协议转译 | CCR 35K Stars |
| ✅ **只做一件事**：告诉你配置哪里有问题，并帮你修 |

### 2.4 护城河

不是"配置格式知识"（公开可查），而是**"什么样的配置是坏的"判断能力** — 需要大量真实案例积累 + 学术研究支撑。

## 四、功能总览

| 命令 | 功能 | 优先级 |
|------|------|--------|
| `ai-lint` | 扫描全部 AI 配置文件 | P0 |
| `ai-lint fix` | 自动修复冗余、冲突 | P0 |
| `ai-lint explain <rule-id>` | 解释规则为何有问题 | P1 |
| `ai-lint stats` | 配置文件健康度趋势 | P1 |
| `ai-lint init` | 生成规范的模板文件 | P2 |
| `ai-lint --ci` | CI 模式（有问题 exit 1） | P2 |

## 六、与竞品的关系

ai-lint 不替代任何现有工具，它在所有工具之上做"诊断层"：

- 用户用 ZCF 初始化环境 → 用 ai-lint 检查配置是否健康
- 用户用 ccusage 看 Token 消耗 → 用 ai-lint 找到浪费的根源
- 用户用 CCR 做模型路由 → 用 ai-lint 检查路由配置是否有冲突

**共生，而非竞争。**
